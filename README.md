# Zenku

A schema-driven full-stack TypeScript application framework. Define your data model once in ZenStack, and Zenku automatically generates a complete admin UI, REST API, and AI agent interface.

## Features

- **Schema-driven UI** — Add a model to `schema.zmodel`, get list/form/detail pages automatically
- **UiConfig** — Customize display labels, column order, form layout, and view modes via `schema/<Model>.ui.ts`
- **Multiple view modes** — Table, Kanban, Calendar, and TreeList views per model
- **Auth** — JWT access token (15 min) + HttpOnly refresh token (7 days), role-based access (ADMIN / USER)
- **Security** — Rate limiting, CORS, secure headers, Zod validation
- **i18n** — English and Traditional Chinese (zh-TW), switchable in the UI
- **File Upload** — Drag-and-drop file/image upload, stored as `Attachment` records
- **AI Integration** — OpenAI-compatible tool API + MCP server for Claude Desktop / Cursor
- **Docker** — Multi-stage build, PostgreSQL in production, SQLite in development
- **Integration Tests** — 44 tests covering auth, CRUD, and access policies

## Stack

| Layer | Technology |
|-------|-----------|
| Runtime | [Bun](https://bun.sh) |
| Backend | [Hono](https://hono.dev) v4 + [ZenStack](https://zenstack.dev) + [Prisma](https://prisma.io) |
| Frontend | React 19 + [TanStack Query](https://tanstack.com/query) v5 + [shadcn/ui](https://ui.shadcn.com) + Tailwind CSS 4 |
| Database | SQLite (dev) / PostgreSQL (prod) |
| Auth | JWT + HttpOnly cookie refresh |

## Development Setup

**Prerequisites:** [Bun](https://bun.sh) >= 1.0

```bash
# Install dependencies
bun install

# Start the server (SQLite dev DB)
cd packages/server && bun run dev

# Start the client (separate terminal)
cd packages/client && bun run dev
```

Client: http://localhost:5173
Server: http://localhost:3001

### First-time setup

```bash
# Generate Prisma + ZenStack (after schema changes)
bunx zenstack generate

# Seed the database
bun run packages/server/src/seed.ts
```

## How to Extend

### Add a new model

1. Add the model to `schema.zmodel` and `schema.production.zmodel`:

```zmodel
model Supplier {
  id        String   @id @default(cuid())
  name      String
  email     String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@allow('all', auth().role == ADMIN)
  @@allow('read', auth() != null)
}
```

2. Add a UI config at `packages/server/schema/Supplier.ui.ts`:

```typescript
import type { UiConfig } from "@zenku/core";

export default {
  label: "Suppliers",
  icon: "Truck",
  list: {
    columns: ["name", "email"],
    searchableFields: ["name"],
    defaultSort: { field: "name", dir: "asc" },
  },
  form: {
    layout: [
      { field: "name", label: "Supplier Name" },
      { field: "email", label: "Email" },
    ],
  },
} satisfies UiConfig;
```

3. Rebuild:

```bash
docker-compose up -d --build
```

The sidebar, list, form, and detail pages are created automatically.

### Add a field to an existing model

1. Add the field in `schema.zmodel` and `schema.production.zmodel`
2. Update the model's `*.ui.ts` to include it in `list.columns` and `form.layout`
3. Rebuild Docker

### Customize a page (Eject)

To take full control of a page beyond what UiConfig supports:

```bash
bun run packages/server/src/eject.ts Product
```

This generates `packages/client/src/pages/ejected/Product{List,Form,Detail}Page.tsx`. Edit these files freely — they take precedence over the generic pages.

> **Note:** Ejected pages are static. After ejecting, schema field changes won't automatically appear in those pages.

### Add a view mode

In the model's `.ui.ts`, add a `kanban`, `calendar`, or `tree` config:

```typescript
// Kanban example
kanban: {
  statusField: "status",
  columns: ["TODO", "IN_PROGRESS", "DONE"],
  cardTitle: "title",
},

// Calendar example
calendar: {
  dateField: "startDate",
  titleField: "title",
},

// Tree example
tree: {
  parentField: "parentId",
  labelField: "name",
},
```

## Configuration

Copy `.env.example` to `.env` and set:

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `JWT_SECRET` | Yes | — | Secret for signing JWTs (min 32 chars) |
| `DATABASE_URL` | Yes | `file:./dev.db` | SQLite path or PostgreSQL URL |
| `PORT` | No | `3001` | Server port |
| `NODE_ENV` | No | `development` | `development` / `production` |
| `CORS_ORIGIN` | No | `http://localhost:5173` | Allowed CORS origin |
| `AI_API_KEY` | No | — | API key for `/api/ai/*` endpoints |

## API Reference

See [`docs/openapi.yaml`](docs/openapi.yaml) for the full OpenAPI 3.1 spec.

### Auth Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/auth/login` | Login, returns access token + sets refresh cookie |
| POST | `/api/auth/register` | Register new user |
| POST | `/api/auth/refresh` | Refresh access token via cookie |
| POST | `/api/auth/logout` | Invalidate refresh token |
| GET | `/api/auth/me` | Current user info |

### CRUD Endpoints

All model endpoints are auto-generated. Replace `{model}` with the lowercase model name (e.g., `product`, `task`).

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/{model}` | List records (pagination, search, sort, filter) |
| POST | `/api/{model}` | Create record |
| GET | `/api/{model}/{id}` | Get record by ID |
| PUT | `/api/{model}/{id}` | Update record |
| DELETE | `/api/{model}/{id}` | Delete record |
| DELETE | `/api/{model}/batch` | Batch delete `{ ids: string[] }` |
| PATCH | `/api/{model}/batch` | Batch update `{ ids: string[], data: {} }` |
| GET | `/api/{model}/export?format=csv` | Export as CSV |

**Query parameters for list:**

| Parameter | Example | Description |
|-----------|---------|-------------|
| `page` | `?page=2` | Page number (default: 1) |
| `pageSize` | `?pageSize=20` | Records per page (default: 10) |
| `search` | `?search=apple` | Full-text search on searchable fields |
| `sortField` | `?sortField=name` | Sort by field |
| `sortDir` | `?sortDir=desc` | Sort direction (`asc` / `desc`) |
| `{field}` | `?isPublic=true` | Filter by field value |
| `include` | `?include=category` | Include relation |

### Other Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/schema` | Schema + UiConfig for all models |
| GET | `/api/health` | Health check + DB status |
| POST | `/api/upload` | Upload file (multipart/form-data) |
| GET | `/api/uploads/{name}` | Serve uploaded file |
| GET | `/api/ai/tools` | AI tool definitions (X-API-Key required) |
| POST | `/api/ai/execute` | Execute AI tool (X-API-Key required) |

## AI Integration

See [`docs/ai-integration.md`](docs/ai-integration.md) for full setup instructions.

**Quick example:**

```bash
# List all tools
curl -H "X-API-Key: $AI_API_KEY" http://localhost:3001/api/ai/tools

# Execute a tool
curl -X POST http://localhost:3001/api/ai/execute \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $AI_API_KEY" \
  -d '{"tool": "list_Product", "arguments": {"take": 5}}'
```

## Docker

docker-compose up -d

docker-compose stop
